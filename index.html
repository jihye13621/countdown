Do a Critique

The current implementation isn't perfect. How could it be improved?

star

Pick an area of improvement to implement.

I focused on the user experience - to make it fun and exciting - how birthdays should be. I added a couple three.js elements (1. emojis and geometries in the header and 2. a 3D model of a rainbow birthday cake on top of the footer). I added an interactive calendar for the user to select their own birthdays and see how many days are left until the birthday. When clicking on the actual birthday, confetti fireworks show up and the text changes accordingly. When clicking on another day, the text resets back again.

palette

What would happen to the current code if the date were changed, the page layout was changed or the format was changed?

If the date were to be in the past, it shows a negative number. In my creation, I started with the CTO's birthday but added the ability for the user to select their own birthday. Their birthday selections show on the countdown as well as the calendar. I had also changed the page layout. I did not have enough time to make it responsive (only looks good on desktop), but I am 100% confident I am able to make it beautiful on responsive mobile and tablet views as well.

code

Does the current implementation follow best practices for HTML, CSS, JS and React?

The code followed a basic Create React App format without major issues. The original code had the date hard-coded and it was in the past. The code could be updated to dynamically show the current year in real time.

Spice it up

Let's be honest, this countdown could be a whole lot more exciting.

timer

What could you do to create a greater sense of urgency?

I made the background of the countdown to be animated gradient rainbow colors. I tried out a flashing/blinking animation at first, but decided that it was distracting and I could not read the numbers well. I also added a progress bar to show how much percentage of the year is left out of 365 days of the year.

filter_vintage

How would you make this countdown more aesthetically pleasing?

I spent some time trying to implement something like this effect but I did not have enough time (given the short deadline of the prompt). I added some background color and rounded edges for more contrast. I would make the site responsive.

brush

Trust your design instincts, and explain how your design choices impact the user experience

I added animated scrolling to the top when clicking on dates on the calendar and animated scrolling to the bottom when clicking on "click birthday below" button since I noticed the website became longer with the calendar that was added. I added some fun looking elements like the floating emojis and 3D cake. The Roboto Serif Google Font was chosen for a classy but bubbly and fun look. I also created the swirly background png in Adoble Illustrator.
